2. FAQ DATABASE STRUCTURE
2.1 Content Schema
Every knowledge entry in the AI Knowledge Command Center follows a standardized schema optimized for search, rendering, and reuse across formats (web, API, export, chatbot).
| Field | Type | Required | Description |
| id | UUID | Yes | Unique content identifier |
| slug | string | Yes | URL-safe identifier (e.g., what-is-retrieval-augmented-generation) |
| title | string | Yes | Primary question or topic title (H1, page title, search result) |
| short_answer | string (280 char) | Yes | One-sentence answer, used in previews, social cards, chatbot responses |
| full_answer | markdown | Yes | Comprehensive answer (500-3000 words), structured with H2/H3 subheadings |
| category_path | string[] | Yes | Hierarchical path, e.g., [generative-ai, rag] |
| difficulty | enum | Yes | BEGINNER, INTERMEDIATE, ADVANCED, EXPERT |
| content_type | enum | Yes | CONCEPT, HOW_TO, COMPARISON, CASE_STUDY, TUTORIAL, OPINION, GLOSSARY |
| tags | string[] | Yes | Flat tags for cross-cutting search (e.g., [llm, retrieval, vector-db]) |
| related_ids | UUID[] | No | Links to related content for graph navigation |
| prerequisites | UUID[] | No | Content that should be read first (learning path support) |
| updated_at | timestamp | Yes | Last substantive update |
| accuracy_score | float | No | AI-generated confidence (0-1), refreshed on review cycle |
| sources | object[] | No | Citations: { url, title, author, date, archive_url } |
| media | object[] | No | Diagrams, videos, code examples: { type, url, caption, alt_text } |
| seo | object | Yes | { meta_title, meta_description, canonical_url, structured_data } |
| engagement | object | Auto | { views, avg_time, helpfulness_votes, share_count } |
| version | int | Yes | Content version number for change tracking |
| status | enum | Yes | DRAFT, REVIEW, PUBLISHED, NEEDS_UPDATE, ARCHIVED |
2.2 Content Types
| Type | Purpose | Typical Length | Example |
| CONCEPT | Explain what something is and why it matters | 800-2000 words | What is Retrieval-Augmented Generation? |
| HOW_TO | Step-by-step guide to accomplish a task | 1000-3000 words | How to Fine-Tune an LLM on Your Data |
| COMPARISON | Side-by-side evaluation of options | 1000-2500 words | Claude vs GPT-4 vs Gemini: Which LLM Should You Use? |
| CASE_STUDY | Real-world implementation with results | 1500-3000 words | How Stripe Uses AI for Fraud Detection |
| TUTORIAL | Hands-on coding or tool walkthrough | 1500-4000 words | Build a RAG Chatbot in 30 Minutes |
| OPINION | Expert analysis of trends or debates | 800-2000 words | Why AI Agents Will Replace SaaS |
| GLOSSARY | Term definition with context | 100-300 words | Tokenization (AI/NLP definition) |
2.3 Difficulty Levels
| Level | Assumed Knowledge | Language Style | Example Topic |
| BEGINNER | No technical background | Plain English, analogies, no code | What is Machine Learning? |
| INTERMEDIATE | Basic programming, stats | Technical but explained, light code | How Does a Transformer Work? |
| ADVANCED | ML/DL experience | Technical depth, code examples, math | Implementing Flash Attention 2 |
| EXPERT | Research-level | Paper references, novel analysis | Scaling Laws and Compute-Optimal Training |
2.4 Linking and Graph Structure
Content entries form a directed knowledge graph via related_ids and prerequisites. This enables learning path generation (topological sort of prerequisites), related content suggestions (graph neighbors + collaborative filtering), topic cluster SEO (pillar pages link to cluster content), and knowledge gap detection (orphan nodes, dead ends).

4.2 Navigation Patterns
Progressive disclosure: User lands on a concept page -> sees related concepts in sidebar -> clicks deeper into technical details -> reaches tutorial or comparison. Each step increases depth.
Difficulty escalation: Content tagged BEGINNER surfaces first. User can toggle to INTERMEDIATE/ADVANCED. Learning paths auto-escalate.
Knowledge graph exploration: Every page shows a mini knowledge graph of related topics. Clicking a node navigates to that content while maintaining graph context.
Search-first: Persistent search bar on every page. Autocomplete shows categories, content, and glossary terms. Semantic search finds conceptually related content even with different wording.
Breadcrumb trail: Full hierarchy displayed: Home > Generative AI > RAG > What is RAG? Enables upward navigation.

5. CONTENT GENERATION WORKFLOW
5.1 Pipeline Overview
1. Topic Identification: Category tree gap analysis + search demand research (Ahrefs, Google Trends) + community questions (Reddit, Stack Overflow, Twitter)
1. Brief Creation: Auto-generate content brief from topic: target keyword, search intent, difficulty level, word count, outline, competitor analysis, unique angle
1. AI Draft: Claude generates first draft from brief. Includes structured answer, code examples, diagrams described, sources cited
1. Human Review: Subject matter expert reviews for accuracy, adds nuance, corrects errors, enhances examples, verifies sources
1. Editorial Polish: Editor ensures consistent voice, readability (Flesch-Kincaid target by difficulty level), SEO optimization, internal linking
1. Media Production: Diagrams, screenshots, code snippets created or generated. Alt text added. Video summary recorded (optional)
1. QA Gate: Fact-check, link verification, accessibility audit, mobile rendering check, SEO checklist
1. Publish + Distribute: Published to site with structured data. Syndicated to newsletter, social, RSS. API updated
1. Performance Monitor: 30/60/90 day review of engagement metrics. Low performers flagged for refresh or merge
1. Evergreen Refresh: Quarterly review cycle. AI flags outdated content (model releases, pricing changes, new research). Human confirms updates

5.3 AI-Assisted Content Operations
- Auto-brief generation: Given a topic and category, Claude generates a complete content brief including outline, target keywords, competitor gaps, and unique angle
- Draft generation: Claude writes first draft from brief, following the content schema and voice guidelines
- Freshness monitoring: Weekly AI scan of all published content against current knowledge. Flags articles where key facts have changed (model releases, pricing, benchmarks)
- Internal linking: AI analyzes new content and suggests links to/from existing articles. Maintains link graph health
- Translation: AI generates translations for top markets (Spanish, Portuguese, Japanese, Hindi, French) with human review for top 50 articles
- Content adaptation: Single source content auto-adapted to newsletter excerpt, social posts, chatbot answer, and API response formats

9.3 Content Principles
1. Accuracy over speed: Never publish unverified claims. Every factual statement should be traceable to a source.
1. Vendor-neutral by default: Compare fairly. Disclose all affiliations. Recommend based on use case, not sponsorship.
1. Difficulty honesty: Label everything clearly. A beginner should never accidentally land in expert-level content without warning.
