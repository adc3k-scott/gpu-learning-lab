Domain 1 — Essential AI Knowledge (38%)
AI vs ML vs Deep Learning?
AI > ML > DL (nested circles). AI = machines mimic intelligence. ML = learn from data. DL = neural nets with many layers.
CPU vs GPU architecture?
CPU: few powerful cores, sequential. GPU: thousands of cores, parallel. GPUs excel at matrix math for AI.
Why GPUs for AI?
AI = massive matrix multiplications, highly parallelizable. GPUs deliver 10-100x speedup. Tensor Cores add mixed-precision acceleration.
Training vs Inference?
Training = build model (days/weeks, huge compute). Inference = use model (low latency, less per-request compute).
Which needs more compute?
Training per run. But at scale, inference can use more total GPU-hours (24/7 serving millions).
What is TensorRT?
Inference optimization engine. Layer fusion, precision calibration (FP32→FP16/INT8), kernel auto-tuning. Faster inference on NVIDIA GPUs.
AI adoption drivers?
(1) GPU compute (2) Data availability (3) Algorithm advances (transformers) (4) Software ecosystem (PyTorch, pre-trained models)
5 major AI industries?
Healthcare, autonomous vehicles, financial services, manufacturing, natural language/LLMs. Also: retail, energy, defense.
NVIDIA software stack?
GPU → CUDA → cuDNN → AI Enterprise (TensorRT, Triton, NIM, RAPIDS) → Frameworks (PyTorch) → Apps (NGC). CUDA is foundation.
What is NGC?
Catalog/registry of GPU-optimized containers, pre-trained models, SDKs. App store for GPU AI software.
What is NIM?
Packages AI models as containerized microservices with REST API. Pull container → get optimized inference endpoint. Simplifies production.
What is Triton?
Open-source inference serving platform. Multiple models, multiple frameworks (PyTorch, TF, TensorRT, ONNX). Batching, scheduling, monitoring.
What is an LLM?
Deep learning model, billions of parameters. Training = thousands of GPUs for weeks. Even inference needs multiple high-memory GPUs (weights in VRAM).
Supervised vs Unsupervised vs RL?
Supervised = labeled data. Unsupervised = finds patterns, no labels. RL = agent learns from rewards/penalties.
Transfer learning?
Take pre-trained model, fine-tune on smaller domain data. Less data needed, less time, good results.
What is RAPIDS?
GPU-accelerated data science libs. cuDF (GPU pandas), cuML (GPU scikit-learn), cuGraph. Speeds up data prep.
Mixed-precision training?
Uses FP16 + FP32. Tensor Cores do FP16 math faster. Master weights in FP32. ~2x speedup, minimal accuracy loss. Blackwell adds FP4/FP8.
AI development lifecycle?
Data collection → prep → model dev → evaluation → deployment (TensorRT, Triton/NIM) → monitoring (drift detection). Iterative.
Containers (Docker) in AI?
Package entire stack (OS, CUDA, frameworks). Portable, reproducible, no dependency conflicts. NGC provides pre-built GPU containers.
What is an epoch?
One complete pass through training data. Many epochs per training run. Too many = overfitting.
What is DGX?
Purpose-built AI supercomputer. DGX H100 = 8x H100 + NVLink/NVSwitch + networking. Building block for DGX SuperPOD.
What is Base Command?
Cluster management for DGX. Job scheduling, resource allocation, user management, monitoring. The OS for a DGX cluster.
What is MLOps?
DevOps for ML. Automates training, testing, deployment, monitoring pipelines. Models degrade (data drift) → need automated lifecycle.
Domain 2 — AI Infrastructure (40%)
What is NVLink?
High-bandwidth GPU-to-GPU interconnect. H100: 900 GB/s bidirectional. Solves communication bottleneck across GPUs.
What is NVSwitch?
Connects all GPUs in a node via NVLink. Fully connected fabric, any-to-any at full bandwidth. Creates unified GPU memory space.
What is InfiniBand?
High-bandwidth, low-latency cluster networking. ConnectX + Quantum switches. 400 Gb/s/port. Minimizes gradient exchange overhead.
InfiniBand vs Ethernet?
IB: 400 Gb/s, ~1µs, native RDMA, best for training. Ethernet: 400 Gb/s, ~5-10µs, needs RoCE, good for inference. IB wins for training.
What is RDMA?
Remote Direct Memory Access. GPU reads/writes remote memory bypassing CPU/OS. Wire-speed gradient exchange. IB = native RDMA. Ethernet = RoCE.
What is BlueField DPU?
Offloads networking, storage, security from CPU. Frees GPUs for AI. Hardware security isolation, zero-trust, storage acceleration.
Scale-up vs Scale-out?
Scale-up = more GPUs per node (NVLink, max ~8). Scale-out = more nodes (InfiniBand). In practice: both.
GPU memory: training vs inference?
Training: params + gradients + optimizer + activations (70B = 8x80GB GPUs). Inference: weights + KV cache. Memory BW often the bottleneck.
Power and cooling for AI?
5-10 kW/node. Air fails above 30 kW/rack. Liquid cooling required. PUE target <1.3. 480V 3-phase. N+1 redundancy.
What is PUE?
Total Facility Power / IT Power. 1.0 = perfect. Avg: 1.58. Modern target: 1.2-1.3. Best liquid: 1.03-1.1.
Air vs Liquid cooling?
Air: up to 30 kW/rack, PUE 1.3-1.6. Liquid: 50-100+ kW/rack, PUE 1.03-1.2. Liquid required for modern GPU clusters.
Direct-to-chip vs Immersion?
Direct-to-chip: cold plates on GPU, rest uses air. Most common. Immersion: entire server in dielectric fluid. Max density, lowest PUE.
Key AI cluster components?
GPU nodes, IB spine-leaf, parallel FS, Quantum switches, management (BCM, DCGM), scheduler (Slurm/K8s), power + cooling.
What is DGX SuperPOD?
NVIDIA reference architecture for large AI clusters. Defines topology, storage, cabling for 20-140+ DGX nodes.
Facility requirements?
20-50+ MW, 30-100 kW/rack density, liquid cooling, 2,000-3,000+ lbs/rack floor load, fiber, physical security, N+1 redundancy.
Storage for AI?
Training: parallel FS (Lustre/GPFS). Checkpointing: high write throughput. NVMe for local scratch. Petabytes, tiered. 100+ GB/s network throughput.
On-prem vs Cloud?
On-prem: full control, high CapEx, lower long-term TCO at sustained use, data sovereignty. Cloud: pay-per-use, fast scale, higher TCO if always-on.
What is spine-leaf?
Two-tier: every leaf to every spine. Max 2 hops, predictable latency, easy horizontal scaling. Standard for AI clusters.
Multi-node training bandwidth?
Each node needs 400 Gb/s+. DGX H100: 8x 400 Gb/s = 3.2 Tb/s total. Network must match compute or GPUs idle.
GPU Direct RDMA + Storage?
GPU Direct RDMA: network adapters ↔ GPU memory (no CPU). GPU Direct Storage: NVMe → GPU memory directly. Both eliminate data copies.
Data vs Model parallelism?
Data: same model on every GPU, different batches, sync gradients. Model: model split across GPUs. Pipeline: stages on different GPUs.
GPU power consumption?
H100 = 700W. DGX H100 = ~10.2 kW. GPU racks 5-10x more power-dense than traditional servers.
What is MIG?
Multi-Instance GPU. Partitions 1 GPU into up to 7 isolated instances. Dedicated compute/memory/BW. A100/H100 only. Maximizes utilization.
GPU hardware selection factors?
Model size (memory), training vs inference, precision needs, scaling, budget, throughput vs latency, software compatibility.
Domain 3 — AI Operations (22%)
What is DCGM?
Data Center GPU Manager. GPU health (temp, power, ECC), diagnostics, policy management. Integrates with Prometheus/Grafana.
GPU metrics to monitor?
Utilization %, memory util, temp (throttle ~83°C), power, ECC errors, SM clock, PCIe/NVLink throughput, memory BW.
What is BMC?
Baseboard Management Controller. Out-of-band: remote power, health, KVM console, firmware. Works when OS is crashed. Uses IPMI.
Kubernetes for AI?
Container orchestration. Deploy/scale inference, manage GPU resources, health checks, rolling updates. + NVIDIA GPU Operator. Best for inference.
NVIDIA GPU Operator?
Automates all NVIDIA software on K8s — drivers, container toolkit, device plugin, DCGM, MIG manager. GPU K8s plug-and-play.
Slurm vs K8s for training?
Slurm: native multi-node GPU jobs (MPI, NCCL), gang scheduling, mature. K8s: better for inference serving. Slurm wins for large training.
Gang scheduling vs Backfill?
Gang: all resources at once (64 GPUs requested = wait for all 64). Required for distributed training. Backfill: small jobs fill idle resources.
GPU virtualization?
MIG: hardware partition, full isolation, A100/H100, up to 7. vGPU: software, time-slicing. MPS: multi-process sharing, less isolation.
Time-slicing vs MIG?
Time-slicing: switches between workloads, shared memory, no isolation. MIG: physically partitioned, dedicated resources, full isolation.
Management protocols?
IPMI/BMC, SNMP, Redfish (REST API), DCGM, Prometheus+Grafana, Syslog/ELK, Ansible/Terraform, Base Command.
Job scheduling considerations?
Priority queues, fair-share, gang scheduling, preemption, resource quotas, topology awareness, checkpointing for long jobs.
ECC memory errors?
Correctable (CE): auto-fixed, increasing count = failing. Uncorrectable (UCE): job fails, drain node, DCGM diagnostics, RMA. Monitor CE trend.
Design 70B LLM cluster?
70B @ FP16 = 140 GB weights, ~560 GB with optimizer. 4-8 DGX H100 (32-64 GPUs), IB NDR, spine-leaf, Lustre 100+ TB, liquid cooling, Slurm, DCGM.
