D1 — System & Server Bring-Up (31%)
SuperPOD deployment sequence?
1. Physical install → 2. OOB/BMC → 3. In-band mgmt → 4. BCM head node → 5. PXE provision nodes → 6. Storage fabric → 7. Compute fabric (IB) → 8. GPU drivers → 9. Validate (HPL/NCCL) → 10. Workload mgr
4 network fabrics?
1. Compute: IB (Quantum XDR/NDR) 2. Storage: IB or Ethernet 3. In-Band Mgmt: Ethernet (SN5610) 4. OOB Mgmt: Ethernet (SN2201, 1G)
Rail-optimized topology?
GPU 0 from every node → same leaf switches. GPU 1 → different set. AllReduce follows rail patterns — stays within one leaf group, minimizes spine hops.
BMC / Redfish?
BMC: embedded controller, OOB mgmt even when OS crashed. Redfish: modern REST API (JSON/HTTPS) replacing IPMI. Secure, scriptable, standardized.
Cable types?
DAC: copper, <3m, lowest latency, intra-rack. AOC: optical, <100m, inter-rack. Transceivers+fiber: >100m, spine-leaf. NDR=OSFP twin-port. XDR=OSFP single.
Firmware to validate?
GPU (InfoROM), BMC, SBIOS, NIC (ConnectX), Switch (MLNX-OS), DPU (BlueField), PSU. All must version-match per NVIDIA release notes.
Cooling by density?
Air: ≤30 kW, PUE 1.3-1.5. Rear-door passive: 30 kW. Rear-door active: 75 kW. CDU: 140+ kW, PUE 1.05. Immersion: 200+ kW, PUE 1.02.
OOB (Out-of-Band) Network?
Separate management network — completely isolated from production data traffic. Carries BMC/IPMI, BCM provisioning, UFM, DCGM, PDU/CDU mgmt. Typically 1GbE (SN2201). If IB or production net goes down, OOB still reaches every device. Your lifeline.
IB generations?
HDR: 200G, QSFP56 (2019). NDR: 400G, OSFP twin, Quantum-2 64-port (2022). XDR: 800G, OSFP, Quantum-X800 144-port (2025).
NVLink generations?
Pascal 80→Volta 300→Ampere 600→Hopper 900→Blackwell 1800→Rubin 3600 GB/s. Links: 4→6→12→18→18→18.
D2 — Cluster Test & Verification (33%)
HPL validation?
High Performance Linpack. Dense linear algebra. Validates compute+memory+interconnect. Target: >80% of theoretical peak. NGC container: hpc-benchmarks. Params: N, NB, P×Q.
NCCL tests?
all_reduce_perf, all_gather_perf, reduce_scatter_perf, broadcast_perf. Validates NVLink BW (intra-node) + IB BW (inter-node). Key metric: bus bandwidth GB/s.
DCGM diag levels?
r=1: seconds (Software, PCIe, Memory, BW). r=2: <2 min (+Diagnostics). r=3: <30 min (+SM/Power/Stress, EUD). r=4: 1-2 hr (+Memtest, Pulse).
11 DCGM plugins?
Deployment, Diagnostic, PCIe BW, GPU Memory, Targeted Power, Targeted Stress, Memtest (11 patterns), Pulse Test, NVBandwidth, Memory BW, NCCL Tests.
Burn-in procedure?
1. DCGM r=4 every node → 2. Single-node NCCL → 3. Single-node HPL → 4. Multi-node NCCL → 5. Multi-node HPL 24-72hr → 6. Monitor ECC/thermal → 7. ClusterKit.
ECC: correctable vs uncorrectable?
CE: auto-fixed, few/day normal, increasing = failing. UCE: job fails, drain + RMA immediately. Volatile = since reset. Aggregate = lifetime. Page retirement = auto.
SHARP?
In-network computing. Reductions done IN switches. 5x AllReduce, 9x Barrier. ~95% network BW. X800: 14.4 TFLOPS. Ops: AllReduce, Reduce, Broadcast, AllGather, ReduceScatter.
DCQCN congestion?
Switch detects congestion → marks ECN bit → receiver sends CNP → sender reduces rate. PFC = pause frames (last resort). Alpha value tracks congestion severity per QP.
GPUDirect RDMA + Storage?
RDMA: network → GPU memory, no CPU. Storage: NVMe → GPU memory, no CPU bounce. Both eliminate data copies. GDS: 3.5x BW, 3.5x lower latency.
Magnum IO stack?
GPUDirect Storage + GPUDirect RDMA + NCCL + NVSHMEM + cuFile. Goal: remove CPU from every data path.
UFM tiers?
Telemetry: 120+ counters/port. Enterprise: +provisioning, validation, SHARP. Cyber-AI: +ML analytics, anomaly detection. SuperPOD uses Enterprise.
Adaptive routing?
Switch picks least-loaded output port dynamically. ARN: congestion notification cascades upstream. Combined with DCQCN. Eliminates hotspots in fat-tree.
D3 — Control Plane (19%)
BCM (Base Command Manager)?
Formerly Bright Cluster Manager. CLI: cmsh. PXE provisioning, image mgmt, monitoring, user mgmt. Now part of NVIDIA Mission Control (BCM + Run:ai).
cmsh device states?
[UP]: monitored, healthy. [DOWN]: monitored, unexpected fail. [CLOSED](UP): intentionally up, no alerts. [CLOSED](DOWN): intentionally down, no alerts.
Node provisioning?
PXE boot → DHCP from head → TFTP kernel → software image from BCM → install. Categories group nodes, assign image + roles (assign slurmclient).
Enroot + Pyxis?
Enroot: rootless container runtime (no Docker daemon). Pyxis: Slurm plugin for --container-image. Use for training. Docker = for K8s/inference.
GPU Operator (K8s)?
9 DaemonSets: Driver, Toolkit, Device Plugin, GFD, DCGM, DCGM-Exporter, MIG Manager, Validator, NFD. ClusterPolicy CRD = single config. Helm install.
DCGM-Exporter?
Containerized. Exposes /metrics for Prometheus. DaemonSet via GPU Operator. Custom fields via CSV. 30s scrape. Grafana dashboards pre-built.
NGC CLI?
ngc config set (API key). ngc registry image pull. Key containers: pytorch, tritonserver, hpc-benchmarks (HPL), nccl-tests.
Slurm for GPUs?
GRES in gres.conf + slurm.conf. --gres=gpu:8. Gang scheduling for distributed training. --exclusive for full-node. Partitions, QOS, fair-share.
Container Toolkit?
nvidia-ctk runtime configure --runtime=docker. Injects GPU devices + driver libs into containers. docker run --gpus all. GPU Operator auto-installs on K8s.
D4 — Troubleshoot & Optimize (12%)
Critical XID errors?
XID 48: double-bit ECC → RMA. XID 79: GPU fell off bus → reseat/replace riser. XID 43: hung GPU. XID 63: row remap fail. XID 94: contained ECC (Hopper+, ok to continue).
GPU replacement procedure?
Drain (scontrol) → verify no jobs → CLOSED(DOWN) in BCM → power off via BMC → swap GPU → boot → nvidia-smi → dcgmi diag -r 3 → NCCL test → undrain.
Performance tuning?
GPU: persistence ON, clocks max, ECC on. CPU: performance governor, disable C-states. Net: adaptive routing, SHARP. Storage: GDS, NVMe scheduler=none. NCCL: env vars.
IB link troubleshooting?
Link down → check cable. Width 1x not 4x → bad cable. High symbol errors → replace cable. BER rising → cable degrading. Speed mismatch → firmware or transceiver.
Storage bottleneck?
GPU util spiky/low = data bottleneck. Check: GDS enabled (gdscheck), Lustre stripe settings, DataLoader workers, local NVMe cache. Target: 40+ GBps/node.
Page retirement?
Auto-retires pages with repeated ECC errors. Single-bit: retired at next reset. Double-bit: retired immediately. High count (>60) = RMA GPU. Check: nvidia-smi -q -d RETIRED_PAGES.
D5 — Physical Layer (5%)
BlueField-3 modes?
NIC Mode: Arm off, standard NIC (ENGINE=1). DPU Mode: Arm active, controls NIC (ENGINE=0, default). Zero Trust: host fully isolated (mlxprivhost). Separated = OBSOLETE.
MIG: single vs mixed?
Single: all GPUs same geometry, standard nvidia.com/gpu resource. Mixed: different per-GPU, resources per-profile (nvidia.com/mig-1g.5gb). ConfigMap config.
DOCA SDK?
Programming framework for BlueField DPUs. CUDA for GPUs = DOCA for DPUs. Networking offload, storage offload, security. Containerized services at scale.
Fabric Manager?
Daemon managing NVSwitch-based GPU fabrics. NVLink topology, GPU routing, health monitoring. Must be running for multi-GPU NVLink. Logs: /var/log/fabricmanager.log.
3D parallelism?
Data: same model, different batches, sync gradients (NCCL). Tensor: split layer across GPUs (NVLink). Pipeline: different layers on different GPUs. Large models use all 3.
Run:ai?
GPU scheduling platform (NVIDIA acquired). Fractional GPUs, quotas, fair-share across teams. Team A borrows Team B's idle GPUs, auto-returns. K8s-native. Part of Mission Control.

B200 vs B300?
B300 = Blackwell Ultra. FP4: 9→15 PFLOPS. HBM3e: 192→288 GB (12-high stacks). TDP: 1,000→1,400W. New: 2x attention acceleration, NVFP4 precision. Shipped Jan 2026.
GB300 vs GB200 NVL72?
Same 72-GPU + 36 Grace rack. GB300: B300 GPUs, 1.1 ExaFLOPS FP4 (vs 0.7), 21 TB GPU memory (vs 13.5), ConnectX-8 (integrated PCIe switch), ~132 kW.
Grace CPU?
72 Arm Neoverse V2 cores. 114 MB L3. LPDDR5X 500 GB/s. Connects to Blackwell GPUs via NVLink-C2C at 900 GB/s (7x PCIe Gen5). Grace handles data preprocessing + memory expansion.
BlueField-4 vs BF-3?
Cores: 16→64 Neoverse V2. Memory: 32→128 GB. Network: 400→800 Gb/s. PCIe: Gen5→Gen6. L3: 8→114 MB. SSD: 128→512 GB. 6x compute. Ships 2026 with Vera Rubin.
Vera Rubin?
Next platform after Blackwell. R100 GPU: 50 PFLOPS FP4 (5x B300). 288 GB HBM4, 22 TB/s BW. NVLink 6: 3,600 GB/s. Vera CPU: 88 Olympus cores. 336B transistors. Production H2 2026.
Spectrum-X vs InfiniBand?
Spectrum-X: AI Ethernet. 5-10μs latency. Best for mid-size clusters, inference, multi-tenant. InfiniBand: <2μs. Best for large-scale training, SuperPOD. IB costs ~2x but lowest latency.
Mission Control?
BCM + Run:ai + BMS. Bright→BCM (2024). Run:ai acquired 2024. Combined = Mission Control. Adds facility monitoring (MQTT), GPU scheduling, fractional GPUs. cmsh still works inside it.
NVSM?
NVIDIA System Management. Hardware-level layer BELOW BCM. Direct node health, GPU telemetry, power mgmt. Stack: NVSM (hardware) → BCM (cluster) → Mission Control (facility).
MGX platform?
Modular Reference Platform. Standardized chassis for OEMs. 100+ configs. Cuts dev cost $2-4M, time 18→6 months. DGX B300 = first MGX-compatible DGX. Any certified OEM can build.
7-chip platform (GTC 2026)?
Vera Rubin GPU + Vera CPU + Groq 3 LPX + BlueField-4 + ConnectX-9 + Spectrum-6 SPX + Space 1. 5 rack systems: GPU rack, CPU rack, LPU rack, storage rack, network rack. All liquid cooled, 45°C hot water, 2-hr install.
Dynamo 1.0 (GTC 2026)?
Open-source inference OS for AI factories. Disaggregates workloads: prefill → GPU, decode → Groq LPU. 7x inference boost on Blackwell (same HW). 700→5,000 tokens/sec. Modules: KVBM (memory), NIXL (GPU-to-GPU), Grove (scaling). Adopted: AWS, Azure, GCP, Oracle.
NemoClaw / OpenClaw (GTC 2026)?
OpenClaw: open-source agent OS — resource mgmt, tool access, LLM integration, scheduling. NemoClaw: NVIDIA production stack = Nemotron models + OpenShell sandbox runtime + privacy router + guardrails. Single-command install. Microsoft Security: 160x threat detection improvement.
DSX Reference Design (GTC 2026)?
Complete AI factory blueprint from NVIDIA. Physical layouts, power specs, cooling specs, network topology. Omniverse DSX digital twin simulates factory before build. DSX Air for design iterations. Jensen: "factor of two in efficiency." Build to spec, not from scratch.
Groq LPU Integration (GTC 2026)?
NVIDIA acquired Groq. GPU handles prefill (heavy reasoning, KV cache). Groq LPU handles decode (fast token output, 500MB on-chip SRAM). Dynamo orchestrates both over Ethernet. 35x tokens-per-watt vs GPU-only. Recommendation: ~25% facility capacity = Groq racks.
Feynman (2028 roadmap)?
Post-Rubin architecture. Feynman GPU + LP40 LPU + Rosa CPU + BlueField-5 + CX-10. Both copper and co-packaged optics (Kyber rack). First gen where LPU is co-designed with GPU from the start.
ADC 3-Layer Power?
