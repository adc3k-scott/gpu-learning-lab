Domain 1: Essential AI Knowledge

23 Questions — Foundations, GPU architecture, NVIDIA software stack, ML concepts

1
What is the primary difference between AI, machine learning, and deep learning?
▾
AI is the broadest concept (machines mimicking human intelligence). Machine learning is a subset of AI where systems learn from data without explicit programming. Deep learning is a subset of ML using neural networks with many layers. Think of it as nested circles: AI > ML > DL.
2
What is the main architectural difference between a CPU and a GPU?
▾
A CPU has a few powerful cores optimized for sequential processing (low latency, complex logic). A GPU has thousands of smaller cores optimized for parallel processing (high throughput, same operation on many data points simultaneously). GPUs excel at the matrix math that AI training and inference require.
3
Why have GPUs become the standard for AI workloads instead of CPUs?
▾
AI workloads involve massive matrix multiplications that are highly parallelizable. GPUs can execute thousands of these operations simultaneously, delivering 10-100x speedup. Tensor Cores add further acceleration for mixed-precision math.
4
What is the difference between AI training and AI inference?
▾
Training = building the model. Feed massive datasets through neural network, adjust weights. Requires enormous compute (days/weeks on GPU clusters). Inference = using the trained model. Feed new data in, get predictions out. Less compute per request but demands low latency.
5
Which requires more GPU compute -- training or inference?
▾
Training requires far more compute per run (days/weeks on hundreds of GPUs). But at scale, inference can consume MORE total GPU-hours because it runs 24/7 serving millions of requests.
6
What is NVIDIA TensorRT?
▾
TensorRT is NVIDIA's inference optimization engine. Takes a trained model and optimizes it -- layer fusion, precision calibration (FP32 to FP16/INT8), kernel auto-tuning, memory optimization. Faster inference with lower latency on NVIDIA GPUs.
7
Key factors driving rapid AI adoption?
▾
(1) Compute power -- GPU acceleration. (2) Data availability -- massive datasets. (3) Algorithm advances -- transformers, attention. (4) Software ecosystem -- PyTorch, TensorFlow, pre-trained models, cloud APIs.
8
5 major industries for AI deployment?
▾
Healthcare (imaging, drug discovery), Autonomous vehicles, Financial services (fraud, trading), Manufacturing (QA, predictive maintenance), Natural language (LLMs, chatbots). Also: retail, energy, telecom, defense.
9
NVIDIA software stack layers?
▾
Bottom to top: GPU Hardware → CUDA → cuDNN → NVIDIA AI Enterprise (TensorRT, Triton, NIM, RAPIDS) → Frameworks (PyTorch, TensorFlow) → Applications (NGC containers, pre-trained models, NIM). CUDA is the foundation.
10
What is NVIDIA NGC?
▾
NGC is a catalog/registry of GPU-optimized containers, pre-trained models, and SDKs. Ready-to-use Docker containers for AI frameworks. Think app store for GPU-accelerated AI software.
11
What is NVIDIA NIM?
▾
NIM packages optimized AI models as containerized microservices with standard REST API. Pull a NIM container, get an inference endpoint already optimized with TensorRT. Simplifies putting AI models into production.
12
What is NVIDIA Triton Inference Server?
▾
Open-source inference serving platform. Serves multiple models from multiple frameworks (PyTorch, TF, TensorRT, ONNX). Handles model management, request batching, scheduling, health monitoring.
13
What is an LLM and why does it need GPU compute?
▾
LLM = deep learning model trained on massive text data (GPT, Llama, etc.). Billions of parameters. Training requires thousands of GPUs for weeks. Even inference requires multiple high-memory GPUs because model weights must fit in GPU memory.
14
Supervised vs unsupervised vs reinforcement learning?
▾
Supervised = labeled data (input → known output). Unsupervised = unlabeled data, finds patterns. Reinforcement = agent learns from environment rewards/penalties (game AI, robotics).
15
What is transfer learning?
▾
Take a pre-trained model, fine-tune on smaller domain-specific dataset. Drastically reduces training time/cost, requires less data, achieves good results with small datasets.
16
What does NVIDIA RAPIDS do?
▾
GPU-accelerated libraries for data science -- GPU versions of pandas (cuDF), scikit-learn (cuML), graph analytics (cuGraph). Speeds up data prep before model training.
17
What is mixed-precision training?
▾
Uses both FP16 and FP32 during training. Tensor Cores do FP16 matrix ops faster. Forward/backward passes in FP16, master weights in FP32. ~2x speedup with minimal accuracy loss. Blackwell adds FP4/FP8.
18
AI development lifecycle stages?
▾
Data collection → Data preparation → Model development → Model evaluation → Deployment (TensorRT, containerize, Triton/NIM) → Monitoring (drift detection, retrain). Iterative cycle.
19
Role of containers (Docker) in AI?
▾
Package entire AI stack (OS, CUDA, frameworks, code) into portable unit. Works same everywhere. Easy scaling, version control, no dependency conflicts. NGC provides pre-built GPU-optimized containers.
20
What is an epoch?
▾
One complete pass through entire training dataset. Training runs many epochs. Each epoch = multiple batches. Batch size = samples processed before weight update. Too many epochs = overfitting.
21
What is NVIDIA DGX?
▾
Purpose-built AI supercomputer. DGX H100 = 8x H100 GPUs with NVLink/NVSwitch, high-bandwidth networking, Base Command software. Building block for DGX SuperPOD.
22
What is NVIDIA Base Command?
▾
Cluster management and orchestration for DGX systems. Job scheduling, resource allocation, user management, cluster monitoring. The OS for a DGX cluster.
23
What is MLOps?
▾
DevOps for ML -- automating training, testing, deployment, monitoring pipelines. Needed because models degrade (data drift), manual deployment doesn't scale, reproducibility requires version control.

Domain 2: AI Infrastructure

24 Questions -- Networking, cooling, cluster design, storage, GPU hardware

24
What is NVLink?
▾
High-bandwidth GPU-to-GPU interconnect. H100: 900 GB/s bidirectional. Solves communication bottleneck when model is split across multiple GPUs.
25
What is NVSwitch?
▾
Chip connecting all GPUs within a node via NVLink into fully connected fabric. In DGX H100, connects all 8 GPUs -- any GPU to any GPU at full bandwidth. Creates unified GPU memory space.
26
What is InfiniBand?
▾
High-bandwidth, low-latency networking for AI clusters. NVIDIA ConnectX + Quantum switches, up to 400 Gb/s/port. GPUs constantly exchange gradients during training -- IB minimizes this overhead.
27
InfiniBand vs Ethernet?
▾
IB: 400 Gb/s, ~1us latency, native RDMA, preferred for training, higher cost. Ethernet: 400 Gb/s, ~5-10us latency, needs RoCE for RDMA, lower cost, good for inference. IB wins for training due to native RDMA.
28
What is RDMA?
▾
Remote Direct Memory Access -- one machine reads/writes another's memory directly, bypassing CPU and OS. Dramatically reduces latency. GPUs exchange gradients millions of times -- RDMA does this at wire speed. IB has native RDMA; Ethernet uses RoCE.
29
What is a DPU / BlueField?
▾
DPU offloads networking, storage, security from CPU. BlueField = NVIDIA's DPU. Frees CPUs/GPUs for AI, provides hardware security isolation, accelerates storage I/O, enables zero-trust.
30
Scale-up vs scale-out?
▾
Scale-up = more GPUs in one node (NVLink, max ~8). Scale-out = more nodes in cluster (InfiniBand). In practice: both. Max GPUs per node, then add nodes.
31
GPU memory for training vs inference?
▾
Training: needs memory for parameters + gradients + optimizer states + activations. 70B model may need 8x80GB GPUs. Inference: holds model weights + KV cache. Memory bandwidth often the bottleneck for LLM inference. Quantization helps.
32
Power and cooling for AI?
▾
Power density 5-10 kW per node. Air cooling fails above 30 kW/rack. Liquid cooling required. PUE target <1.3. 480V 3-phase for efficiency. N+1 redundancy.
33
What is PUE?
▾
Total Facility Power / IT Equipment Power. 1.0 = perfect. Industry avg: ~1.58. Modern target: 1.2-1.3. Best liquid-cooled: 1.03-1.1. Lower = less waste.
34
Air vs liquid cooling?
▾
Air: up to ~30 kW/rack, PUE 1.3-1.6, lower upfront. Liquid: 50-100+ kW/rack, PUE 1.03-1.2, higher upfront but lower ongoing. Liquid effectively required for modern GPU clusters.
35
Direct-to-chip vs immersion?
▾
Direct-to-chip (CDU): cold plates on GPU chips, rest uses air. Most common enterprise. Immersion: entire server submerged in dielectric fluid. Maximum heat removal, highest density, lowest PUE.
36
Key components of AI cluster?
▾
GPU compute nodes, high-speed network (IB/Ethernet spine-leaf), storage (parallel FS), network switches (Quantum/Spectrum), management plane (BMC, Base Command, DCGM), job scheduler (Slurm/K8s), power+cooling.
37
What is DGX SuperPOD?
▾
NVIDIA reference architecture for large-scale AI clusters. Defines how to connect multiple DGX systems with IB networking, shared storage, management. Scales 20-140+ nodes. Specifies exact topology, storage, cabling.
38
Facility requirements for GPU infrastructure?
▾
Power capacity (20-50+ MW), power density (30-100 kW/rack), liquid cooling, floor loading (2,000-3,000+ lbs/rack), fiber-ready networking, physical security, clean agent fire suppression, N+1 redundancy.
39
Storage for AI workloads?
▾
Training data: high-throughput parallel FS (Lustre, GPFS). Checkpointing: high write throughput. IOPS: NVMe SSDs for local scratch. Capacity: petabytes, tiered storage. Network: storage fabric must keep up with GPU demand (100+ GB/s).
40
On-prem vs cloud?
▾
On-prem: full control, high CapEx, lower long-term TCO at sustained use, data sovereignty. Cloud: pay-per-use, minutes to scale, higher TCO if always-on, limited customization. On-prem for sustained; cloud for bursty.
41
What is spine-leaf topology?
▾
Two-tier: every leaf connects to every spine. Predictable low latency (max 2 hops), high bandwidth (parallel paths), easy horizontal scaling. Standard for AI clusters.
42
Networking bandwidth for multi-node training?
▾
Each node needs 400 Gb/s+. DGX H100 has 8x 400 Gb/s ConnectX-7 (3.2 Tb/s total). Network bandwidth must be proportional to compute capability -- otherwise GPUs idle waiting for gradient sync.
43
GPU Direct RDMA and GPU Direct Storage?
▾
GPU Direct RDMA: network adapters access GPU memory directly (no CPU). Critical for training scalability. GPU Direct Storage: NVMe transfers to GPU memory directly. Both eliminate unnecessary data copies.
44
Data parallelism vs model parallelism?
▾
Data parallel: same model on every GPU, different data batches, sync gradients. Works when model fits one GPU. Model parallel: model split across GPUs (different layers). Required when too large. Pipeline parallel: stages on different GPUs.
45
GPU power consumption?
▾
H100 = 700W TDP. DGX H100 (8 GPUs) = ~10.2 kW. Traditional server: 300-500W. GPU racks are 5-10x more power-dense. 1,000 GPUs = 1-2 MW just for GPUs.
46
What is MIG?
▾
Multi-Instance GPU. Partitions one GPU into up to 7 isolated instances with dedicated compute, memory, bandwidth. A100/H100 only. Run multiple inference workloads on one GPU with full isolation. Maximizes utilization.
47
Factors for GPU hardware selection?
▾
Model size (memory needs), training vs inference, precision requirements, scaling needs, budget, throughput vs latency, software compatibility.

Domain 3: AI Operations

13 Questions -- Monitoring, scheduling, Kubernetes, virtualization, cluster design

48
What is DCGM?
▾
NVIDIA Data Center GPU Manager. GPU health monitoring (temp, power, ECC errors), diagnostic testing, policy-based management, integrates with Prometheus/Grafana. Core GPU monitoring tool.
49
What GPU metrics to monitor?
▾
GPU utilization %, memory utilization, temperature (throttle at ~83C), power draw, ECC errors (correctable trend, uncorrectable = replace), SM clock frequency, PCIe/NVLink throughput, memory bandwidth utilization.
50
What is BMC?
▾
Baseboard Management Controller. Embedded controller for out-of-band management: remote power, health monitoring, KVM console, firmware updates, event logging. Works even when OS is crashed. Uses IPMI.
51
Kubernetes for AI?
▾
Container orchestration. Deploys/scales inference containers, manages GPU resources, health checks, auto-restart, rolling updates. Integrates with NVIDIA GPU Operator. Best for inference serving.
52
NVIDIA GPU Operator?
▾
Automates all NVIDIA software on Kubernetes -- GPU drivers, container toolkit, device plugin, DCGM, MIG manager, GFD. Installs as containers. Makes GPU K8s plug-and-play.
53
Why Slurm over K8s for training?
▾
Slurm: native multi-node GPU jobs (MPI, NCCL), gang scheduling, lightweight, mature priority scheduling. K8s better for inference serving. Slurm better for large training jobs.
54
Gang scheduling vs backfill?
▾
Gang: all resources allocated simultaneously (64 GPUs requested = wait until all 64 available). Required for distributed training. Backfill: smaller jobs run on idle resources while large job waits.
55
GPU virtualization approaches?
▾
MIG: hardware partitioning, full isolation, A100/H100, up to 7 instances. vGPU: software virtualization, time-slicing, VMware/KVM, good for VDI. MPS: multi-process sharing, less isolation, works on all GPUs.
56
Time-slicing vs MIG?
▾
Time-slicing: GPU switches between workloads, shared memory, no isolation, simpler. MIG: physically partitioned, dedicated compute/memory, full isolation, A100/H100 only.
57
Data center management protocols?
▾
IPMI/BMC (out-of-band), SNMP (network monitoring), Redfish (REST API for IPMI), DCGM (GPU monitoring), Prometheus+Grafana (metrics), Syslog/ELK (logs), Ansible/Terraform (automation), Base Command (DGX clusters).
58
GPU cluster job scheduling considerations?
▾
Priority queues, fair-share scheduling, gang scheduling, preemption, resource quotas, job dependencies, multi-node topology awareness, checkpointing for long jobs.
59
GPU ECC memory errors in production?
▾
Correctable (CE): detected/corrected by hardware, few per day normal, increasing count = failing memory. Uncorrectable (UCE): causes job failure, drain node, run DCGM diagnostics, RMA GPU. Monitor CE trend as early warning.
60
Design an AI cluster for 70B LLM training?
▾
70B params at FP16 = ~140 GB weights. With optimizer: ~560 GB total. Spec: 4-8 DGX H100 nodes (32-64 GPUs), IB NDR 400 Gb/s between nodes, spine-leaf topology, parallel FS (Lustre/GPFS 100+ TB, 50+ GB/s), liquid cooling (~82 kW for 8 nodes), Slurm for scheduling, DCGM for monitoring.
