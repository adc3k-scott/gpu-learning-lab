# Multi-Location / Franchise Playbook

**One discovery call. One master config. Deployed to every location. Franchise owners don't want 15 separate projects — they want one decision that scales.**

---

## The Problem

Franchise owners and multi-location operators have 3-50 locations running the same business. Same vertical, same workflows, same pain points — just replicated across sites.

They don't need 15 discovery calls. They don't need 15 custom configs. They need one system that clones cleanly and handles site-specific differences (staff names, hours, inventory, local phone numbers).

Traditional IT vendors charge per-location as if each one is a brand new project. That's a ripoff. We don't do that.

---

## Solution: Master Config + Clone Deploy

### Step 1: Discovery (One Call)
- Single discovery call with the franchise owner or operations manager.
- Map the standard workflow across all locations.
- Identify what varies per location (staff, hours, inventory, local contacts).
- Build the **master config** — one template that covers 90% of every location.

### Step 2: First Location Install (Full Install)
- **3 hours** — full playbook install at the flagship or busiest location.
- This becomes the reference install. Every config decision, integration, and workflow gets documented.
- Train the location manager. Get their feedback.
- Lock the master config.

### Step 3: Clone Deploy (Each Additional Location)
- **1 hour per location** — config clone + site-specific customization.
- What gets cloned: AI agent config, workflow rules, integrations, NemoClaw policies, dashboard layout.
- What gets customized: Staff names, business hours, local phone number, location-specific inventory, local vendor contacts.
- Train local staff (30 min — they already know the brand's workflow, just need to learn the AI tools).

## Centralized Dashboard

The franchise owner gets a **single dashboard** that shows all locations on one screen.

### What the Owner Sees
- **Location grid**: Status of every location (online/offline, active tasks, alerts).
- **Aggregate metrics**: Total calls handled, total appointments booked, total revenue influenced — across all locations.
- **Per-location drill-down**: Click any location for its individual dashboard.
- **Comparative analytics**: Which location is performing best? Which one needs attention?
- **Unified client communications**: One inbox for all location notifications, escalations, and AI agent alerts.

### What Each Location Manager Sees
- Their location's dashboard only.
- No visibility into other locations (unless owner grants access).
- Same interface as a single-location install — no extra complexity for local staff.

## Sales Approach: Top-Down

### Step 1: Find the Decision Maker
- Go through franchise **corporate** or the **multi-location owner** first.
- Don't pitch individual location managers — they can't say yes to a system-wide deployment.
- LinkedIn, local business associations, franchise expos, trade shows.

### Step 2: Pitch the Math
- "You have 10 locations all doing the same thing. Why pay for 10 separate AI projects?"
- "One decision, one config, deployed everywhere in 2 weeks."
- "Your competitors are doing this location by location. You can do it all at once."

### Step 3: Pilot One Location
- Always start with one location as a pilot (full install, full price).
- Prove the value. Get the location manager to confirm results.
- Then roll to remaining locations with the clone deploy model.

### Step 4: Roll Down
- Schedule clone deploys in batches (3-5 per week is comfortable for one installer).
- Each location gets 1 hour of installer time + 30 min staff training.
- Owner sees locations light up on the dashboard one by one.

## Installer Notes

- **Franchise installs are the highest-value work you can do.** One sale = weeks of install work + massive recurring override.
- **The first location install is critical.** This is where you prove the system works. Treat it like a demo for the other 9-49 locations.
- **Document everything on location 1.** Every config choice, every integration detail, every site-specific note. This becomes your clone template.
- **Clone deploys are fast but not autopilot.** Each location has quirks — different staff, different hours, sometimes different POS versions. Budget the full hour.
- **The owner is your real client.** Keep them updated on rollout progress. Send a summary after each batch of installs. They're betting their whole operation on this.
