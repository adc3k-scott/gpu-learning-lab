The Seven Layers

Every layer builds on the one below it. Click to expand.

LAYER 0
Physical Transport
Fiber • Satellite • The wire in the ground
▶

Everything rides on this. No Layer 0 = no network, no compute, nothing.

Dedicated Fiber

Dark fiber between Willow Glen and MARLIE I (60 mi). Management traffic, monitoring, backup control. Sub-millisecond latency over dedicated strand.

Site Fiber

Fiber runs from wetland sites to nearest corridor (I-10, Hwy 90, Hwy 182). Connects edge sites back to Willow Glen spine. Max 5 mi to existing fiber corridor.

Satellite

Starlink / VSAT for offshore rigs and barges. 20-40ms latency — fine for edge inference and periodic sync. Not suitable for InfiniBand fabric.

ROV parallel: Scott ran subsea fiber optic multiplexers for 25 years. Most aircraft mechanics don't know fiber — Scott does. Layer 0 is infrastructure he already understands.
LAYER 1
Power Architecture
Gas • Solar • Battery • Grid • Diesel
▶

No power = no compute. The 3-layer power stack is locked across all sites.

Layer 1 Primary

Nat Gas + Solar + Battery

24/7 baseload. Henry Hub pricing (cheapest in the US). Solar offset during peak sun. Battery buffer for transients.

Layer 2 Backup

Grid / LUS

Never primary. Backup only. Connected but not depended on.

Layer 3 Emergency

Diesel Gensets

On-site fuel. Pipeline-independent. Learned from Hurricane Ida — gas pipeline was down for weeks.

Willow Glen is different: 4-layer Mini Grid (solar, nat gas, grid, biodiesel) + future green hydrogen from electrolysis. Former 2,200 MW Entergy station — power infrastructure already exists.
LAYER 2
Cooling & Thermal
EC-110 • Immersion • Heat Recovery
▶

SITE VIABILITY = POWER + COOLING. This is half the equation.

Immersion Cooling

GPU boards submerged in EC-110 dielectric fluid. Non-toxic, non-conductive, non-flammable. Fluid runs at 60-70°C. No server fans. PUE 1.03.

External Rejection

Dry coolers reject heat to ambient. Zero water consumption. No evaporative cooling towers.

Heat Recovery (Phase 2)

ORC electricity recovery (ElectraTherm/Infinity Turbine, 3-5% efficiency). Solar thermal boost to 85-90°C doubles output. Direct thermal sales to co-located customers.

Closed-loop system: EC-110 absorbs GPU heat → heat exchanger → dry cooler rejects to ambient (or ORC captures electricity, or thermal customer uses heat). Cooled fluid returns to tank. Same as ROV deck pack with seawater heat exchanger.
LAYER 3
Compute Fabric
InfiniBand • GPUs • DGX SuperPOD
▶

The reason everything else exists. GPU-to-GPU communication at microsecond latency.

Compute Network

InfiniBand (NVIDIA Quantum switches). GPU-to-GPU during training and inference. Highest bandwidth, lowest latency of all four networks. This is the spine.

Storage Network

High-speed Ethernet or dedicated storage fabric. Moves training data and model weights. A starved GPU is a wasted GPU.

NVLink / NVSwitch

GPU-to-GPU inside a single node at 900 GB/s. Internal bus — like crank position to injector timing. No external network needed.

Fat-tree topology: Pods are leaf nodes on the InfiniBand spine. Scale wide (add pods) or punch in (inject at bottleneck). All spine switches live at Willow Glen — the fabric grows from there.
LAYER 4
Network Intelligence
SmartNICs • DPUs • Software-Defined
▶

The ECU of the AI factory. Reads sensors, processes data, adjusts in real time.

ConnectX SmartNICs

The sensors. Sit on every server, reading network traffic, measuring latency, reporting congestion. Like O2 sensors in an engine management system.

BlueField DPUs

The ECU. Process network data and make routing/security decisions without bothering the GPU. Offload infrastructure work so GPUs only do AI compute.

Software-Defined Networking

Closed-loop tuning. The system reads its own performance data and adjusts — not static configuration, adaptive. Like a closed-loop fuel map that learns.

ECM parallel: FuelTech ECU reads sensors → processes data → adjusts fuel/timing in real time. ConnectX reads traffic → BlueField processes → SDN adjusts routing in real time. Same closed-loop control, different scale.
LAYER 5
Management Plane
Base Command • In-Band • Out-of-Band
▶

Two management networks — one for software, one for hardware emergencies.

In-Band Management

Software-level: monitoring, updates, job scheduling, telemetry. How NVIDIA Base Command Manager talks to every node — assigning workloads, collecting performance data.

Out-of-Band Management

Hardware-level: works even when the server OS is crashed. Power cycling, BIOS access, remote KVM. Like having a physical hand on every machine. Separate network for security.

Base Command Manager

NVIDIA's orchestration platform. Runs at Willow Glen on the cluster itself. Manages the entire compute fabric — scheduling, monitoring, scaling. The brain.

Willow Glen = Primary. MARLIE I = Mirror. Base Command runs at Willow Glen because it needs to be on the InfiniBand fabric. MARLIE I gets a read-only mirror — Scott watches everything from Lafayette.
LAYER 6
Operations & Autonomy
NOC • Drones • AI Ops • MARLIE I
▶

The human and AI operations layer. Where decisions get made and executed.

Primary NOC (Willow Glen)

On-site operations center at the flagship. Staff monitor the full fabric — compute, power, cooling, security. 24/7 operations.

Backup NOC (MARLIE I)

Scott's war room in Lafayette. Full monitoring mirror. If Willow Glen NOC goes down, MARLIE I takes over. Also R&D staging — test new pod configs here first.

Autonomous Ops

Drones patrol sites (KLFT hub). Robots maintain pods. AI manages compute scheduling, power optimization, thermal management. Minimal permanent human presence at remote sites.

ROV control room model: The captain doesn't stand on the drill floor. He's in the control room watching everything on screens. MARLIE I is Scott's control room. Willow Glen is the drill floor.
